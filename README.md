# рџЏ¦ LoanSentry - Intelligent Credit Risk Assessment

An end-to-end AI-powered fintech platform combining Machine Learning, Retrieval-Augmented Generation (RAG), and Explainable AI to predict loan default risk and provide policy-grounded explanations.

---

## рџ“ё Overview

LoanSentry analyzes loan applications using a weighted ensemble of 4 ML models, explains predictions using SHAP values, retrieves relevant policy documents via semantic search, and answers credit risk questions using an LLM (Groq/Llama 3.1).

The entire project was built and tested locally. Training scripts are provided so anyone can train their own models and build their own RAG knowledge base from scratch using their own documents.

---

## вњЁ Features

- **Risk Assessment** - Predict default probability with a weighted ensemble (LR + RF + XGBoost + Neural Network)
- **SHAP Explainability** - Visualize top features driving each prediction
- **RAG Knowledge Base** - Semantic search over custom policy documents with LLM-powered Q&A
- **Risk Trend Simulation** - Simulate how changing one feature affects default probability
- **Applicant Comparison** - Compare two applicant profiles side by side with AI interpretation
- **Global Explainability** - SHAP summary plots across synthetic profiles with AI insights
- **Model Performance** - View accuracy, F1, ROC-AUC across all models
- **PDF Report** - Download a full risk assessment report
- **Logs & Feedback** - Track predictions and user feedback over time

## вљ™пёЏ Setup

### 1. Clone the repo

```bash
git clone https://github.com/arjunmehra05/LoanSentry.git
cd LoanSentry
```

### 2. Create a virtual environment

```bash
python -m venv venv
venv\Scripts\activate       # Windows
source venv/bin/activate    # Mac/Linux
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

This installs everything needed for both the app and the training scripts.

### 4. Get the dataset

This project was trained on the **LendingClub loan dataset** available on Kaggle.

1. Create a free account at [kaggle.com](https://www.kaggle.com)
2. Download the dataset: [wordsforthewise/lending-club](https://www.kaggle.com/datasets/wordsforthewise/lending-club)
3. Extract and note the path to `accepted_2007_to_2018Q4.csv`
4. Update `DATASET_PATH` at the top of `training/01_data_preprocessing.py`

> **Note:** The dataset is ~2GB. The training scripts use 500K rows by default which is sufficient for good results. You can adjust `NROWS` in `01_data_preprocessing.py` if needed.

### 5. Train the models

Run the scripts in order - each script reads from the `output/` folder written by the previous one:

```bash
python training/01_data_preprocessing.py   # ~5 mins
python training/02_ml_models.py            # ~10 mins
python training/03_neural_network.py       # ~20 mins (GPU recommended)
python training/04_ensemble.py             # ~15 mins
```

After all scripts complete, copy the generated files from `output/` to `models/`:

```
models/

в”њв”Ђв”Ђ model_rf.pkl
в”њв”Ђв”Ђ model_xgb.pkl
в”њв”Ђв”Ђ model_nn.keras
в”њв”Ђв”Ђ scaler.pkl
в”њв”Ђв”Ђ feature_names.pkl
в”њв”Ђв”Ђ xgb_threshold.pkl
в”њв”Ђв”Ђ nn_threshold.pkl
в”њв”Ђв”Ђ ensemble_weights.pkl
в””в”Ђв”Ђ ensemble_threshold.pkl
```

> These scripts were originally developed on Google Colab. If running locally, a GPU is recommended for the neural network step but is not required.

### 6. Build your RAG knowledge base

The `knowledge_base/` folder is where you add your own `.txt` documents. These can be anything relevant to your domain - credit policy guidelines, risk frameworks, lending regulations, or any domain knowledge you want the system to retrieve from.

**Tips for good RAG quality:**
- Write in full sentences and paragraphs - avoid bullet points or label:value formatted files
- Each file should cover one topic clearly
- Aim for files between 300 and 800 words each

Once you have added your documents, build the FAISS index:

```bash
python training/05_rag_pipeline.py
```

Copy the generated files to the `rag/` folder:

```
rag/
в”њв”Ђв”Ђ faiss_index.index
в””в”Ђв”Ђ chunks.pkl
```

### 7. Set up environment variables

Create a `.env` file in the root folder:

```
GROQ_API_KEY=your_groq_api_key_here
```

Get a free Groq API key at [console.groq.com](https://console.groq.com). The app works without it but AI explanations and knowledge base Q&A will fall back to rule-based responses.

### 8. Run the app

```bash
streamlit run app.py
```
