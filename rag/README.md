# RAG Knowledge Base Directory

This folder should contain the Retrieval-Augmented Generation (RAG) index files generated by the RAG pipeline script.

## Required Files

Place the following files here after building the RAG knowledge base:

- `faiss_index.index` - FAISS vector index for semantic search
- `chunks.pkl` - Pickled document chunks

## Instructions

1. Add your policy documents (.txt files) to the `knowledge_base/` folder
2. Run the RAG pipeline: `python training/05_rag_pipeline.py`
3. Copy the generated files from `output/` to this `rag/` folder

## Tips for Good RAG Quality

- Write documents in full sentences and paragraphs (avoid bullet points)
- Each file should cover one topic clearly
- Aim for 300-800 words per document
- Include relevant domain knowledge, policies, and guidelines
